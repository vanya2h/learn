## Highlights

First public release of the Dashboard — a self-hostable, open-source study tracker built on React Router v7, Hono, Better Auth, and Prisma. It lets learners follow AI-generated curricula, track daily progress, and build custom learning programs powered by Claude.

## Changes

### New Features

**Core platform**
- Full-stack SSR app with React Router v7 (framework mode) and a Hono API layer (`src/server/app.ts`, `app/routes/api.ts`)
- Email/password authentication via Better Auth with 1-year sessions (`src/server/auth.ts`)
- PostgreSQL database via Prisma with `PrismaPg` adapter — progress models: `TaskCompletion`, `DailyActivity`, `Specialization`, `AppSetting` (`prisma/schema.prisma`)
- Docker support for self-hosting

**Learning & curriculum**
- AI-powered curriculum builder using Claude — generates structured programs with topics, tasks, and hands-on exercises (`src/data/curriculums/`)
- Custom program creation: users can build and manage their own programs
- Skills system with unlock progression — only earned skills are displayed
- Activity heatmap derived from `DailyActivity` task completion counts
- Sticky sidebar for in-session navigation

**AI / LLM**
- Streaming LLM responses managed with RxJS (`feat: use rxjs for stream management`, `feat: refactor study page with rxjs streams`)
- Per-user LLM usage limits with enforcement modal (`feat: add llm usage limit`, `feat: add rate limit modal`)
- Prompts hidden from end users

**Profile & UX**
- CV parsing to pre-fill user profile (`feat: add cv parsing`)
- Improved profile UX and lazy auth flow
- Localization support (i18n)
- shadcn/ui component library with custom border-sliced Vercel-style design system

**Infrastructure**
- Turborepo monorepo with pnpm workspaces
- CI pipeline with build and type-check jobs
- User role column for future access control (`feat: add user role column`)
- Hono RPC client for fully type-safe browser-to-API calls (`src/lib/apiClient.ts`)
